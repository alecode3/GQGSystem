Visión del Sistema

¿Qué es GQG System?

Sistema que extiende la facturación de contado a crédito automatizado. Parametriza vencimientos regulares e irregulares mediante Triggers PostgreSQL.

React 19 TypeScript TailwindCSS Vite Supabase JS PostgreSQL Triggers SQL PERT / CPM

REQUERIMIENTOS DEL CLIENTE

CO
Contado: Cuotas únicas canceladas en la misma fecha de la factura.
CR
Crédito: Múltiples cuotas — Regular (30/60/90 días) o Irregular (45/75 días con %)

Implementación Técnica

PostgreSQL con lógica de negocio centralizada en Triggers SQL. Arquitectura de lógica concentrada en BD para consistencia transaccional.

🗄️ Diagrama de BD

CLIENTES
PK id
nombres, apellidos
documentonro
VENTAS
PK id
FK clienteid
FK plazoid
totalfactura
PLAZOS
PK id
cuotas, irregular
PLAZO_DETALLES
PK id
cuota, %, dias
CUENTAS_COBRAR
PK id
importe, cobrado
vence, estado
VENTAS → CUENTAS_COBRAR (1:N) · PLAZOS → PLAZO_DETALLES (1:N)

⚡ Trigger ins_ventas

CREATE TRIGGER ins_ventas AFTER INSERT ON VENTAS FOR EACH ROW BEGIN -- Validar coherencia tipo doc vs plazo IF doctipoid <> plazotipoid THEN SIGNAL SQLSTATE '45000' SET MESSAGE_TEXT = 'Tipo doc y plazo no coinciden.'; END IF; -- Prorrateo con TRUNCATE importecuota = TRUNCATE(NEW.totalfactura / nCuotas, dec); ultimacuota = NEW.totalfactura - (importecuota * (nCuotas - 1)); -- Insertar cuotas en CUENTAS_COBRAR WHILE irow <= nCuotas DO INSERT INTO CUENTAS_COBRAR ... END WHILE; END;
✓ Coherencia tipo doc vs plazo ✓ TRUNCATE sin pérdida decimal ✓ Regular e irregular

Pruebas funcionales, de integración y validación de reglas de negocio para garantizar la calidad del sistema.
